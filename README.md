## 這是什麼？

「王董」是一個以台灣傳統製造業老闆為原型設計的 **Claude Code Skill**。

你扮演顧問，王董扮演你最難搞的潛在客戶。每次正式提案前，先讓他來挑毛病，比真實提案更殘酷、更安全。

**王董是誰：**
- 台中五金零件廠老闆，做了三十幾年，員工六十幾人
- 兒子剛回來接班，但帳還是他在簽
- 對顧問的基本態度：「你們都是來收錢的」
- 決策邏輯：面子 > 關係 > 錢 > 效率 > 邏輯

**適合誰用：**
- MarTech 顧問 / 數位轉型顧問
- 業務、PM，需要向傳統產業提案的人
- 任何需要在台灣傳產場景「練兵」的人

## 示範對話

```
你：王董您好，我是做 MarTech 顧問的，今天想跟您介紹一套客戶管理系統……

王董：客戶管理系統。這樣喔。多少錢？

你：方案從月費 3 萬起……

王董：什麼？這樣要這麼貴。我老客戶都認識了，用 LINE 就好，
      我幹嘛花這個錢。對岸找人做才這個價的三分之一。

你：王董，我理解您的考量。這套系統的價值在於……

王董：（打斷）你有做過像我們這種的嗎？五金零件的？
      不一樣的，你做過大公司的那些，跟我們這種不一樣。
```

說「暫停」：王董跳出角色，分析你剛才哪裡失分、可以怎麼應對。

## 安裝方法

### 方式一：Git Clone（推薦）

```bash
# 全域安裝（所有專案都能用）
git clone https://github.com/aster-life/taiwan-client-skill ~/.claude/skills/wang-dong
```

### 方式二：手動下載

1. 下載此 repo 的所有檔案
2. 放到 `~/.claude/skills/wang-dong/` 目錄

### 確認安裝

```bash
ls ~/.claude/skills/wang-dong/
# 應看到：SKILL.md  persona.md  work.md  README.md
```

**重要**：安裝後需要**完全重啟 Claude Code**（⌘Q 關閉再重開），新 Skill 才會被載入。

## 使用方法

重啟 Claude Code 後，輸入：

```
/wang-dong
```

王董會開口說：
> 「你好，我是王董。你說你是做 MarTech 的，什麼叫 MarTech？」

接著你就開始提案。

### 可用指令

| 你說的話 | 效果 |
|---------|------|
| `/wang-dong` | 啟動王董，開始提案演練 |
| `開始` | 讓王董進入刁難模式 |
| `暫停` | 王董跳出角色，分析你剛才的表現 |
| `繼續` | 回到角色繼續演練 |
| 直接提案 | 王董直接回應你的提案內容 |

## 王董的刁難模式

王董內建五個台灣在地的典型情境：

**1. 還沒聽完就先砍價**
> 「多少錢？……對岸做才三分之一。」

**2. 同業比爛**
> 「XX 工廠也試過，你知道他說什麼嗎？」

**3. 政府補助依賴**
> 「有補助我考慮看看，沒補助你說服我為什麼我要自己出錢。」

**4. 二代衝突**
> 「我兒子說好。但帳是我在簽。」

**5. 特殊性封殺術**
> 「你說的這些，是一般公司的做法。我們公司很不一樣。」

## 如何讓王董越來越像真實客戶

**追加真實素材**：把你遇到的真實客戶對話（Email、會議紀錄、Line 訊息截圖）貼給 Claude，說：
> 「這是我真實客戶的對話，幫我更新王董的 persona」

**對話中糾正**：如果王董的反應不夠像：
> 「他不會這樣說，他比較會說 XXX」
Claude 會記錄到 Correction 層，立即生效。

**版本控制**：每次更新前，備份一份：
```bash
cp -r ~/.claude/skills/wang-dong ~/.claude/skills/wang-dong-v1
```

## 背後設計邏輯

這個 Skill 基於 [colleague.skill](https://github.com/titanwings/colleague-skill) 的雙層架構設計：

| 層 | 檔案 | 內容 |
|---|---|---|
| Work Skill | `work.md` | 王董的決策框架、談判策略、產業知識 |
| Persona | `persona.md` | 五層人格：硬規則 → 身份 → 表達 → 決策 → 人際 |

執行邏輯：`Persona 決定態度 → Work Skill 執行決策 → 以王董的口吻輸出`
