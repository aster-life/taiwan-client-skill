## 這是什麼？

這是一個以**台灣傳統企業老闆**為原型設計的 **Claude Code Skill**，用來模擬提案現場最難搞的那個人。

你扮演顧問，AI 扮演你的潛在客戶——然後挑你的毛病、問你的底線、拖延你的進度。每次正式提案前，先讓他來一輪，比真實提案更殘酷、更安全。

**預設客戶：王董**
- 台中五金零件廠老闆，做了三十幾年，員工六十幾人
- 兒子剛回來接班，但帳還是他在簽
- 對顧問的基本態度：「你們都是來收錢的」
- 決策邏輯：面子 > 關係 > 錢 > 效率 > 邏輯

**也可以是任何人：** 透過啟動時的第三個問題，你可以把預設的王董換成你實際面對的客戶——南部食品廠老太太、北部中型貿易商、科技業採購主管，任何人都行。

**適合誰用：**
- MarTech 顧問 / 數位轉型顧問
- 業務、PM，需要向傳統產業提案的人
- 任何需要在台灣 B2B 場景「練兵」的人

## 啟動流程

輸入 `/wang-dong` 後，AI 會先以**正常身份**問你三個問題，收到答案後才進入角色：

```
在王董開口之前，讓我先了解你的情況，這樣他才能針對你來刁難。

1. 你的身份是什麼？
   例如：顧問、業務、PM、創業者（以及你的專業領域）

2. 你今天想提案的內容是什麼？
   一句話描述即可，例如：「導入 CRM 系統」「數位行銷服務」「ERP 升級方案」

3. 你要模擬的客戶背景是什麼？（可選）
   例如：「南部傳統食品廠老闆，做了二十年，很在乎成本」
         「北部中型貿易商，第二代接班，想現代化但預算保守」
   若跳過，使用預設背景：台中五金零件廠，三十幾年，兒子接班。
```

收到回答後，AI 進入角色：

> 「你好，我是王董。你說你是做＿＿的，什麼叫＿＿？」

## 示範對話

```
[AI 以正常身份詢問]
在王董開口之前，讓我先了解你的情況……
1. 你的身份是什麼？
2. 你今天想提案的內容是什麼？
3. 你要模擬的客戶背景？（可選）

[你]
1. MarTech 顧問
2. 導入客戶管理系統（CRM）
3. 跳過，用預設王董

[王董進入角色]
你好，我是王董。你說你是做 MarTech 的，什麼叫 MarTech？

[你]
王董您好，MarTech 就是用科技工具來管理行銷流程……
今天想跟您介紹一套客戶管理系統，可以幫您系統化管理老客戶的往來記錄。

[王董]
客戶管理系統。這樣喔。多少錢？

[你]
方案從月費 3 萬起……

[王董]
什麼？這樣要這麼貴。我老客戶都認識了，用 LINE 就好，
我幹嘛花這個錢。對岸找人做才這個價的三分之一。

[你]
王董，我理解您的考量。這套系統的價值在於……

[王董]
（打斷）你有做過像我們這種的嗎？五金零件的？
不一樣的，你做過大公司的那些，跟我們這種不一樣。
```

說「**暫停**」：AI 跳出角色，分析你剛才哪裡失分、可以怎麼應對。
說「**繼續**」：回到角色繼續演練。

## 安裝方法

### 方式一：Git Clone（推薦）

```bash
# 全域安裝（所有專案都能用）
git clone https://github.com/aster-life/taiwan-client-skill ~/.claude/skills/wang-dong
```

### 方式二：手動下載

1. 下載此 repo 的所有檔案
2. 放到 `~/.claude/skills/wang-dong/` 目錄

### 確認安裝

```bash
ls ~/.claude/skills/wang-dong/
# 應看到：SKILL.md  persona.md  work.md  README.md
```

**重要**：安裝後需要**完全重啟 Claude Code**（⌘Q 關閉再重開），新 Skill 才會被載入。

## 使用方法

重啟 Claude Code 後，輸入：

```
/wang-dong
```

AI 先問你三個問題，確認你的角色、提案內容、以及要模擬的客戶背景，然後進入角色開始刁難。

### 可用指令

| 你說的話 | 效果 |
|---------|------|
| `/wang-dong` | 啟動，AI 先問情境設定問題 |
| `暫停` | 跳出角色，分析你剛才的表現 |
| `繼續` | 回到角色繼續演練 |
| 直接提案 | 客戶直接回應你的提案內容 |

## 台灣在地刁難情境（預設王董版）

**1. 還沒聽完就先砍價**
> 「多少錢？……對岸做才三分之一。」

**2. 同業比爛**
> 「XX 工廠也試過，你知道他說什麼嗎？」

**3. 政府補助依賴**
> 「有補助我考慮看看，沒補助你說服我為什麼我要自己出錢。」

**4. 二代衝突**
> 「我兒子說好。但帳是我在簽。」

**5. 特殊性封殺術**
> 「你說的這些，是一般公司的做法。我們公司很不一樣。」

## 自定義客戶（讓模擬更貼近真實）

### 方式一：啟動時的 Q3 設定

啟動時回答第三個問題，描述你實際面對的客戶：

```
你要模擬的客戶背景是什麼？
→ 「南部食品廠老闆娘，做醬料二十幾年，兒子讀完台大要接班，
    她表面說讓他做，但所有採購還是她決定。在意成本和老員工感受。」
```

AI 會以這個描述**完全取代**預設的王董身份，生成符合該客戶的語氣、知識背景與刁難方式。

### 方式二：對話中糾正

如果 AI 的反應不夠像真實客戶：

> 「他不會這樣說，他比較會說 XXX」

AI 會立即記錄並調整，在後續對話中生效。

### 方式三：貼入真實對話素材

把你遇到的真實客戶對話（Email、會議紀錄）貼進來：

> 「這是我真實客戶的對話，幫我更新客戶 persona」

## Fork 成你自己的版本

這個 Skill 是通用模板。你可以 fork 後修改成你最常遇到的客戶類型：

| 目標客戶 | 改法 |
|----------|------|
| 科技業 CTO | 修改 `persona.md` Layer 1 身份，`work.md` 改為「只看數字、不相信質化結論」 |
| 傳產接班二代 | 調整 Layer 1 為夾在舊思維與新觀念之間的第二代 |
| 政府採購科長 | 改為「流程第一、創新第二、風險零容忍」的評估框架 |

**改法：**

```
persona.md  → Layer 1 替換客戶身份，Layer 0 / 3 / 4 / 5 為通用邏輯不用動
work.md     → 替換「預設」區塊的知識背景與對提案類型的預設立場
```

## 檔案結構

```
wang-dong/
├── SKILL.md        ← Skill 入口，Claude Code 載入用（含場景確認流程）
├── persona.md      ← 刁難客戶五層人格模板（Layer 0-5）

```

**執行邏輯：** `SKILL.md 收集情境 → Persona 決定態度 → Work Skill 執行決策 → 以客戶口吻輸出`

## 背後設計邏輯

此 Skill 基於 [colleague.skill](https://github.com/titanwings/colleague-skill) 的雙層架構，擴展為五層 Persona 系統：

| 層 | 標記 | 內容 |
|---|---|---|
| Layer 0 | 【通用】 | 所有刁難客戶的核心行為邏輯（不可覆蓋） |
| Layer 1 | 【可覆蓋】 | 客戶身份（預設王董，Q3 可完全替換） |
| Layer 2 | 【依 L1 調整】 | 說話方式、口頭禪、語氣 |
| Layer 3-5 | 【通用框架】 | 決策邏輯、人際行為、雷區（通用，細節依身份具體化） |
